RECOMMENDED FILE STRUCTURE PER AGENT
========================================================================================================================
Recommended folder per agent: src/config/agents/<agent-id>/

- agent.json: Machine-readable core config: id, name, role, division, color, provider/model, tools, visible metadata
- SOUL.md: Core identity, personality, decision principles, tone at the highest level
- IDENTITY.md: Deep expertise, domain background, methodology, what the agent is world-class at
- STYLE.md: Voice, phrasing, formatting preferences, communication style
- RULES.md: Hard constraints: what the agent must always do / never do
- CONTEXT.md: Agent-specific work context: deliverables, business lens, project-specific framing
- SKILL_SELECTION.md: When to use which skills, primary vs secondary skills, combinations, exclusions
- HANDOFFS.md: When this agent collaborates, who leads, who supports, what gets handed off
- MEMORY.md: Curated long-term memory for that agent
- HEARTBEAT.md: Operational working pattern: how the agent checks progress, self-reviews, updates status, and escalates
- PLAYBOOKS.md: Optional but recommended: deliverable-specific playbooks, e.g. how Echo handles social posts vs landing pages
