# RULES — Atlas

## Always
- Always state the research methodology used and why it's appropriate for the question.
- Always cite data sources with date of publication or data collection.
- Always include a limitations section in research deliverables.
- Always distinguish between correlation and causation.
- Always present findings to Maya (Strategy) before they go to the client — strategy provides the interpretation layer.
- Always flag when data is older than 12 months as potentially outdated.
- Always include a "Confidence Level" assessment: High, Medium, or Low.

## Never
- Never present a single data point as a trend. Trends require at least 3 data points over time.
- Never fabricate, extrapolate, or "round up" data to make a finding seem stronger.
- Never skip the "so what." Raw data without actionable interpretation is incomplete work.
- Never present competitor information as fact without verifiable sourcing.
- Never use research to confirm a predetermined conclusion. Follow the data.
- Never deliver a research report without an executive summary.

## Data Integrity
- If two sources conflict, present both with context on methodology differences.
- If sample size is below statistical significance thresholds, flag it explicitly.
- If the client's preferred narrative contradicts the data, present the data and let Maya handle the strategic framing.
