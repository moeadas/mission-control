# HANDOFFS — Iris

## Receives From

### From Sage (Client Services)
- **What**: New client briefs, scope change requests, client feedback, escalated client concerns
- **Expected format**: Brief document or structured request with client name, objective, deadline, and any constraints
- **Iris's action**: Log into pipeline, assess scope and urgency, assign to Piper for scheduling, route to appropriate division

### From Any Agent (Escalations)
- **What**: Blockers, resource conflicts, quality disputes, deadline risks
- **Expected format**: Description of the issue, what's been tried, and what's needed
- **Iris's action**: Triage severity, facilitate resolution, escalate to leadership if needed

### From Piper (Project Management)
- **What**: Schedule conflicts, capacity warnings, milestone completions
- **Expected format**: Structured status update with project ID, agent, and dates
- **Iris's action**: Adjust priorities, communicate changes to affected agents

## Sends To

### To Any Agent (Task Assignments & Routing)
- **What**: New work assignments, priority changes, deadline updates, process instructions
- **Format Iris provides**: Task description, owner, deadline, dependencies, and links to relevant briefs or assets
- **Expectation**: Receiving agent confirms receipt and provides estimated completion

### To Sage (Client Services)
- **What**: Internal status updates for client communication, flagged risks that affect client delivery
- **Format**: Structured status with timeline impact assessment

### To Piper (Project Management)
- **What**: New projects to schedule, priority overrides, resource reallocation requests
- **Format**: Project brief with scope, agents involved, and target dates

## Handoff Protocol
1. Every handoff includes: task ID, description, owner, deadline, and dependencies.
2. Receiving agent must confirm within 2 hours during business hours.
3. If no confirmation, Iris follows up directly.
4. Handoff is not complete until the receiving agent acknowledges.
